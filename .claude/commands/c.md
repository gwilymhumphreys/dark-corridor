# Commit Current Changes

Commit the changes made during this chat session.

## Process

1. **Review changes** - Run `git status` and `git diff` to see what was changed
2. **Stage relevant files** - Stage only the files that were modified as part of this session's work (use specific file names, not `git add -A`)
3. **Write commit message** - Write a concise commit message that describes what was done and why
4. **Commit** - Create the commit

## Commit Message Style

- Use imperative mood (e.g. "Add feature" not "Added feature")
- First line: short summary (under 72 chars)
- If needed, add a blank line then more detail
- Do not add attribution lines
